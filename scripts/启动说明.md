# eBPF AI Agent 启动说明

## 快速启动

### 前台运行

```bash
./start.sh --log-level debug
```

### 后台运行（Daemon 模式）

```bash
./start.sh --daemon --log-level info
```

### 停止服务

```bash
./stop.sh
```

## 命令行选项

| 选项 | 说明 | 示例 |
|------|------|------|
| `-l, --log-level LEVEL` | 日志级别：debug, info, warn, error | `--log-level debug` |
| `-k, --api-key KEY` | Minimax API 密钥 | `--api-key sk-xxxxx` |
| `-K, --api-key-env VAR` | 从环境变量读取 API 密钥 | `--api-key-env MINIMAX_API_KEY` |
| `-c, --config FILE` | 配置文件路径 | `--config /path/to/config.yaml` |
| `-d, --daemon` | 后台守护进程模式 | `--daemon` |
| `-h, --help` | 显示帮助信息 | `--help` |

## 使用示例

### 1. 设置 API 密钥（环境变量方式，推荐）

```bash
export MINIMAX_API_KEY="sk-xxxxx"
./start.sh --daemon
```

### 2. 直接传入 API 密钥

```bash
./start.sh --api-key "sk-xxxxx" --daemon
```

### 3. 使用自定义配置文件

```bash
./start.sh --config /etc/ebpf-ai-agent/config.yaml --daemon
```

### 4. 调试模式

```bash
./start.sh --log-level debug
```

## API 密钥加密存储

API 密钥支持 AES-256-GCM 加密存储，避免明文泄露。

### 加密 API 密钥生成

```go
// 使用 Go 代码生成加密密钥和加密后的 API key
package main

import (
    "ebpf-ai-agent/pkg/config"
)

func main() {
    // 1. 生成随机加密密钥（32字节 = AES-256）
    key, _ := config.GenerateEncryptionKey()
    println("加密密钥:", key)  // 保存到环境变量或安全存储

    // 2. 加密 API key
    encrypted, _ := config.EncryptAPIKey("your-api-key-here", key)
    println("加密后:", encrypted)
}
```

### 配置加密密钥

将生成的密钥设置为环境变量：

```bash
export ENCRYPTION_KEY="你的加密密钥base64编码"
```

### 配置文件示例（加密）

```yaml
# ebpf-ai-agent configuration
log_level: info
encrypted_api_key: "加密后的API密钥密文"
encryption_key: "${ENCRYPTION_KEY}"
```

## 智能过滤层

内置模式匹配过滤层，减少不必要的 AI API 调用：

| 分类 | 说明 | 处理方式 |
|------|------|----------|
| **白名单** | 明显正常的命令（如 `ls`, `cat /var/log/*.log`, `ps` 等） | 直接跳过，不消耗 AI token |
| **黑名单** | 明显恶意行为（如读取 `/etc/shadow`、可疑的 reverse shell 等） | 直接告警，跳过 AI 分析 |
| **灰名单** | 不确定的命令（如 `curl` 下载、`python` 执行等） | 提交 AI 进行风险分析 |

### 黑名单检测示例

以下行为会被直接标记为恶意并告警：
- 访问 `/etc/shadow`
- 远程脚本 pipe to bash (`wget ... | bash`)
- Netcat 反向 shell (`nc -e /bin/bash ...`)
- 密码暴力破解工具 (`hydra`, `john`)
- 可疑的 chmod 权限修改

## 配置文件

首次启动时自动创建 `config.yaml`，内容示例：

```yaml
# ebpf-ai-agent configuration
log_level: info
minimax_api_key: "${MINIMAX_API_KEY}"
```

API 密钥通过环境变量 `MINIMAX_API_KEY` 注入，不会明文存储在配置文件中。

## 日志文件

- 前台运行：输出到标准输出
- 后台运行：输出到 `logs/ebpf-ai-agent.log`

日志示例：
```
[AGGREGATED] pid=1234 duration=5s cmds=10 unique_files=8 first=/bin/ls last=/tmp/script.sh
[FILTER] result=greylisted reason=curl command
[RISK REPORT] {"pid":1234,"risk_level":"low",...}
```

## PID 文件

后台运行时 PID 保存到 `/var/run/ebpf-ai-agent.pid`

## 注意事项

1. 首次使用需要先编译项目：
   ```bash
   go build -o ebpf-ai-agent ./cmd
   ```

2. 确保 eBPF 探测点有权限访问（需要 root 或 CAP_SYS_ADMIN 能力）

3. 使用 `rsyslog` 或 `logrotate` 管理日志文件大小
