# eBPF 探针编译指南

## 探针代码说明

本探针挂载到 Linux 内核的 `sched_process_exec` Tracepoint，用于捕获进程执行（exec）事件。

**核心流程：**

1. 从 Ring Buffer 申请内存
2. 提取当前进程 PID（通过 `bpf_get_current_pid_tgid()`）
3. 提取父进程 PPID（从 Tracepoint 上下文）
4. 提取可执行文件路径（通过 `bpf_probe_read_kernel_str`）
5. 提交事件到 Ring Buffer，供用户态消费

**CO-RE 实现方式：**

- 使用 `vmlinux.h`（由 bpftool 从内核 BTF 生成）定义结构
- 使用 libbpf 风格 API（不是 BCC）
- 使用 `bpf_probe_read_kernel_str` 代替直接指针解引用

## 生成 vmlinux.h

BTF（BPF Type Format）是内核提供的调试信息，用于 CO-RE。

```bash
# 检查 BTF 是否可用
ls -la /sys/kernel/btf/vmlinux

# 如不可用，尝试生成
sudo bpftool btf dump file /sys/kernel/btf/vmlinux format c > /tmp/vmlinux.h
```

**注意：** vmlinux.h 必须与目标内核版本匹配。不同内核版本的 `trace_event_raw_sched_process_exec` 结构可能不同。

## 项目内编译（通过 go generate）

项目使用 Go + cilium/ebpf，构建时自动调用 bpf2go：

```bash
# 生成 eBPF Go 绑定
go generate ./...

# 完整构建
CGO_ENABLED=0 go build -o ebpf-ai-agent ./cmd
```

### 手动编译 probe.c（调试用）

```bash
cd bpf/

# 编译为 eBPF 目标文件
clang -target bpf -O2 -g \
    -D__TARGET_ARCH_x86 \
    -I. \
    -c probe.c \
    -o probe.o

# 验证
llvm-readelf -S probe.o
llvm-objdump -d probe.o
```

### 交叉编译（ARM64）

```bash
# 安装交叉编译器
sudo apt install gcc-aarch64-linux-gnu

# ARM64 编译
clang -target bpf -O2 -g \
    -D__TARGET_ARCH_arm64 \
    -I. \
    -c probe.c \
    -o probe_arm64.o
```

## 内核结构体兼容性

不同内核版本的 Tracepoint 结构可能不同。当前探针适配以下结构：

```c
struct trace_event_raw_sched_process_exec {
    struct trace_entry ent;
    u32 __data_loc_filename;  // 间接访问
    pid_t pid;
    pid_t old_pid;            // 6.8+ 内核用 old_pid
};
```

如遇编译错误，检查 `/tmp/vmlinux.h` 或 `/sys/kernel/btf/vmlinux` 中的实际结构体定义。

---

## Windows 本地编译限制

eBPF 代码**无法在 Windows 本地编译**，原因：

1. `vmlinux.h` 来自 Linux 内核 BTF 数据
2. `bpf_helpers.h`、`bpf_tracing.h` 是 Linux 专有头文件
3. `clang -target bpf` 编译器只能在 Linux 环境运行

**解决方案：** 在 Linux 虚拟机或云服务器中完成编译。

## 常见问题

### 1. `vmlinux.h not found`

```bash
# 重新生成
sudo bpftool btf dump file /sys/kernel/btf/vmlinux format c > bpf/vmlinux.h
```

### 2. `BPF_CORE_READ` 找不到

确认使用的是 libbpf 风格，不是 BCC 风格。当前代码使用 `bpf_probe_read_kernel_str`。

### 3. `permission denied` (eBPF 加载)

云服务器通常禁用 unprivileged eBPF。需要在支持 eBPF 的环境（如本地虚拟机、物理机、路由器）上测试。
